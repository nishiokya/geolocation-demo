⚡ Geolocation API - Cache Control with maximumAge

このチュートリアルについて:
Geolocation APIのmaximumAgeオプションを使用したキャッシュ制御について学びます。
パフォーマンスの最適化とバッテリー消費の削減に役立ちます。

📊 maximumAge とは？

maximumAgeは、キャッシュされた位置情報を使用可能な最大期間（ミリ秒）を指定します。

設定値 動作 使用ケース
0 キャッシュを使用せず、常に新しい位置情報を取得 リアルタイムの高精度な位置情報が必要な場合
5000 5秒以内のキャッシュがあれば再利用 頻繁に更新されるアプリケーション
60000 1分以内のキャッシュがあれば再利用 一般的なマップアプリケーション
300000 5分以内のキャッシュがあれば再利用 おおよその位置情報で十分な場合
Infinity どれだけ古くてもキャッシュを使用 初回のみ位置情報が必要な場合（非推奨）
⚠️ 注意:
maximumAgeを大きく設定すると、古い位置情報が返される可能性があります。 アプリケーションの要件に応じて適切な値を設定してください。

📝 サンプルコード

maximumAge の使用例 
// パターン1: キャッシュを使用しない（常に最新）
navigator.geolocation.getCurrentPosition(
    successCallback,
    errorCallback,
    {
        enableHighAccuracy: true,
        timeout: 10000,
        maximumAge: 0  // キャッシュを使用しない
    }
);

// パターン2: 30秒以内のキャッシュを使用
navigator.geolocation.getCurrentPosition(
    successCallback,
    errorCallback,
    {
        enableHighAccuracy: false,
        timeout: 5000,
        maximumAge: 30000  // 30秒（30000ms）
    }
);

// パターン3: 5分以内のキャッシュを使用
navigator.geolocation.getCurrentPosition(
    successCallback,
    errorCallback,
    {
        enableHighAccuracy: false,
        timeout: 3000,
        maximumAge: 300000  // 5分（300000ms）
    }
);

// 実践例: キャッシュの使用状況を確認
let lastTimestamp = null;

navigator.geolocation.getCurrentPosition(
    function(position) {
        const currentTimestamp = position.timestamp;
        
        if (lastTimestamp === currentTimestamp) {
            console.log('🔵 キャッシュされた位置情報を使用');
        } else {
            console.log('🟢 新しい位置情報を取得');
            lastTimestamp = currentTimestamp;
        }
    },
    errorCallback,
    { maximumAge: 60000 }
);
