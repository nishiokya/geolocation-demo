⏱️ Geolocation API - Timeout Control

このチュートリアルについて:
Geolocation APIのtimeoutオプションを使用したタイムアウト制御について学びます。
適切なタイムアウト設定で、ユーザー体験を向上させる方法を理解できます。

⏰ timeout とは？

timeoutは、位置情報の取得を待つ最大時間（ミリ秒）を指定します。 この時間内に位置情報が取得できない場合、TIMEOUTエラー（code: 3）が発生します。

設定値 動作 使用ケース メリット/デメリット
Infinity タイムアウトなし（デフォルト） 位置情報が必須で待てる場合 ✅ 確実に取得
❌ 無限に待つ可能性
3000 3秒でタイムアウト クイックレスポンスが必要 ✅ 高速
❌ 失敗しやすい
5000 5秒でタイムアウト 一般的なモバイルアプリ ✅ バランス良好
⚠️ 環境により失敗
10000 10秒でタイムアウト 位置精度を重視する場合 ✅ 成功率高い
❌ 少し遅い
30000 30秒でタイムアウト 初回の位置取得 ✅ 高確率で成功
❌ 長い待ち時間
⚠️ 注意:
タイムアウトを短くしすぎると、GPS初期化前にタイムアウトが発生する可能性があります。 特に初回アクセス時やコールドスタート時には、長めのタイムアウトを設定することを推奨します。
💡 ベストプラクティス:
• 初回取得: 10〜30秒の長めのタイムアウト
• リアルタイム追跡: 5〜10秒の中程度のタイムアウト
• クイックチェック: 3〜5秒の短めのタイムアウト
• タイムアウト発生時はリトライ処理を実装する

📝 サンプルコード

timeout の使用例とリトライ処理 
// パターン1: 基本的なタイムアウト設定
navigator.geolocation.getCurrentPosition(
    successCallback,
    errorCallback,
    {
        enableHighAccuracy: true,
        timeout: 5000,        // 5秒でタイムアウト
        maximumAge: 0
    }
);

// パターン2: タイムアウトとエラーハンドリング
navigator.geolocation.getCurrentPosition(
    function(position) {
        console.log('位置情報取得成功', position.toJSON());
    },
    function(error) {
        if (error.code === error.TIMEOUT) {
            console.error('タイムアウトしました');
            // リトライ処理など
        }
    },
    { timeout: 10000 }
);

// パターン3: リトライ機能付き
function getLocationWithRetry(maxRetries = 3, timeout = 5000) {
    let retryCount = 0;
    
    function attempt() {
        navigator.geolocation.getCurrentPosition(
            function(position) {
                console.log('成功!', position.toJSON());
            },
            function(error) {
                if (error.code === error.TIMEOUT && retryCount < maxRetries) {
                    retryCount++;
                    console.log(`リトライ ${retryCount}/${maxRetries}`);
                    // 次回はタイムアウトを延長
                    timeout = timeout * 1.5;
                    attempt();
                } else {
                    console.error('位置情報の取得に失敗', error);
                }
            },
            { timeout: timeout }
        );
    }
    
    attempt();
}

// パターン4: プログレス表示付き
function getLocationWithProgress(timeout = 10000) {
    const startTime = Date.now();
    
    // プログレスバー更新
    const interval = setInterval(() => {
        const elapsed = Date.now() - startTime;
        const progress = (elapsed / timeout) * 100;
        updateProgressBar(progress);
    }, 100);
    
    navigator.geolocation.getCurrentPosition(
        function(position) {
            clearInterval(interval);
            updateProgressBar(100);
            console.log('取得完了');
        },
        function(error) {
            clearInterval(interval);
            console.error('エラー', error);
        },
        { timeout: timeout }
    );
}
